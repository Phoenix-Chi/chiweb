# 技术栈重构基线审计

## 审计时间

- 日期：2026-03-12
- 范围：`src/app`、`src/components`、`src/server`、`package.json`

## 当前现状（重构前）

- 后端存在双轨实现：`Express + Mongoose` 与 Next Route Handler 并存。
- 数据层为 MongoDB 模型，缺少 Prisma schema 与数据库迁移规范。
- 上传链路存在接口契约不一致：前端依赖 `fileId`、`/api/media/clean`、`DELETE /api/media/:id`，后端实现不完整。
- UI 层以 `Antd + inline style` 为主，Tailwind 仅有基础接入。

## 重构目标

- 主链路迁移到 `tRPC + Zod + Prisma + PostgreSQL`。
- API 入口统一在 Next App Router。
- 上传链路实现上传、删除、批量清理三类契约，并与节点保存流程打通。
- 前端优先迁移关键页面到 Tailwind/shadcn 风格组件，保留复杂页面的渐进替换策略。

## 风险与缓解

- 数据迁移风险：Mongo 与 PostgreSQL 模型差异较大。  
  缓解：先落地 Prisma schema，再提供脚本化迁移入口。
- 接口兼容风险：前端已依赖旧 REST 路由。  
  缓解：保留 `/api/nodes`、`/api/media` 兼容层，内部调用 tRPC caller。
- 页面回归风险：时间轴页逻辑复杂。  
  缓解：先迁移主页/登录页组件，不一次性重写时间轴交互。

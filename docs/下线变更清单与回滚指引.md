# 下线变更清单与回滚指引

## 目的

本文档用于记录本次“旧 Express/Mongoose 路径下线”的具体改动，以及遇到紧急问题时的回滚策略。

## 下线变更清单

### 1) 运行路径下线

- 删除旧独立服务入口：`src/server/index.ts`
- 删除旧 REST 路由实现：`src/server/routes/node.ts`、`src/server/routes/media.ts`
- 删除旧 Mongoose 模型：`src/server/models/Node.ts`
- 删除旧文件清理工具：`src/server/utils/deleteMediaFiles.ts`

### 2) 依赖清理

已从 `package.json` 移除以下依赖（及相关类型包）：

- `express`
- `mongoose`
- `cors`
- `multer`
- `dotenv`
- `moment`
- `react-calendar-timeline`
- `mongodb`

保留：

- `@trpc/*`、`zod`、`prisma`、`@prisma/client`

### 3) 前端与样式清理

- 删除旧演示页面：`src/app/timeline.tsx`
- 删除全局样式中的 `react-calendar-timeline` 引入

### 4) 兼容程序移除

- 删除 `scripts/migrate-mongo-to-postgres.ts`
- Prisma schema 中移除 `legacyMongoId` 兼容字段

## 验证基线

每次变更后至少执行：

```bash
npm run type-check
npm run lint
npm run build
```

## 回滚指引

### 场景 A：API 逻辑异常（推荐快速回滚）

1. 回滚到最近稳定提交（由维护者在本地执行）。
2. `npm install` 还原依赖。
3. 重新执行质量门禁，确保恢复可运行状态。

### 场景 B：数据库写入异常

1. 立即停止相关写入操作。
2. 基于业务表（`Node`、`Media`）和数据库快照定位受影响数据。
3. 必要时从迁移前快照恢复。

## 注意事项

- 生产环境变更前，必须先做数据库全量备份。
- 结构性改动优先在预发布环境验证，并记录验证结果。
